# 1.2 패치 노트

S(이동 정지) 키 추가 시점부터의 변경 사항입니다.

## 신규 조작 / 기능
- **화면 잠금 단축키**: 기본 `Y`로 카메라 잠금을 켜고 끕니다. `CAMERA_LOCK` 액션을 `state.keybinds`와 **조작키 변경** 모달에 추가했으며, 재지정된 키를 카메라 잠금 옵션 라벨에 표시합니다. 입력 시 스킬 로그에 잠금/해제 상태를 표시합니다.
- **Y 미작동 원인·수정**: `index.html`이 `app.js`를 버전 없이 불러와 브라우저가 이전 JavaScript를 재사용할 수 있었습니다. `app.js?v=1.2.6`와 `styles.css?v=1.2.6`으로 자산 URL을 변경해 수정본을 강제 로드합니다. 캐시된 이전 HTML과 새 JavaScript가 섞여도 초기화가 중단되지 않도록 라벨 참조도 방어합니다.
- **근접 평타 히트스캔**: 근접 평타의 `spawnBullet()` 투사체를 제거했습니다. 공격 시전시간 종료 프레임에 고정된 사거리 내 대상에게 즉시 피해·피격 이펙트·타격음을 적용합니다. 과전하 원거리 평타의 투사체는 유지합니다.
- **캐릭터 대사 단일 채널**: 패시브·Q/W/E/R 대사를 `activeVoice` 하나로 통합했습니다. 대사가 재생 중이면 이후 요청된 대사는 큐에 넣거나 기존 대사를 끊지 않고 폐기하며, 현재 대사가 종료되거나 재생 실패한 뒤에만 다음 대사를 허용합니다.
- **이동 사운드 무봉제 루프**: 일반 이동·공격 이동으로 실제 좌표가 변하는 동안 `걷기.mp3`를 루프 재생합니다. 시작·정지에 0.12초 볼륨 페이드를 적용하고, 정지 시 재생 위치를 유지해 다음 이동에서 같은 첫 음이 반복되지 않도록 했습니다. 백스텝·볼트 러시·D 돌진 등 `state.dash` 스킬 이동과 순간이동에는 재생하지 않습니다.
- **전하 충전 초기 상태 수정**: 최초 전하 0 상태에서 전하 충전 버튼이 `charge`만 5로 바꾸고 `chargeDecay`를 0으로 남겨 다음 프레임에 즉시 전하가 소멸하던 원인을 수정했습니다. 일반 상태에서는 `addCharge()`로 전하와 5초 지속시간을 함께 설정하고, 과전하 상태에서는 탄환과 과전하 지속시간을 재충전합니다.
- **평타 공격속도 상한**: 평타 입력 수락 시 `state.nextBasicAttackAt`과 `state.cooldowns.A`를 `1 / 공격속도`로 잠급니다. 우클릭·Shift+클릭·공격 이동·스킬 후 자동 공격을 연속 입력해도 잠금 종료 전 추가 평타가 시작되지 않습니다. 쿨타임 초기화 모드는 평타 공격속도 잠금과 평타 쿨다운 표시를 제거하지 않습니다.
- **전체 오디오 50% 감쇠**: `MASTER_AUDIO_GAIN = 0.5`를 음성 대사와 모든 스킬·적중 사운드에 공통 적용합니다.
- **스킬 시전 사운드 매핑**: 근거리 Q=`Q_근접.mov`, 원거리 Q=`Q_원거리.mp3`, W1=`W1.mp3`, W2=`W2.mp3`, R1=`R1.mp3`, R2=`R2.mp3`, F=`F.mp3`, D=`D.mp3`를 시전 시 재생합니다.
- **패시브 상태 사운드**: 전하 만충으로 과전하 진입 거리의 파란 원이 나타날 때 `P_돌입.mp3`, 과전하 진입 시 `P_과전하.mp3`, 과전하 종료 시 `P_종료.mp3`를 재생합니다. 과전하 루프는 2개의 오디오 트랙을 사용해 끝 0.25초와 시작 0.25초를 교차 페이드하며, R2 과전하 재충전에서는 재시작하지 않고 이어서 재생합니다. 종료·초기화에서 두 트랙을 정지합니다.
- **R1 일반 오디오 롤백**: R1 전용 Web Audio API 모노 다운믹스·`AudioContext` 활성화·폴백 처리를 제거했습니다. R1도 다른 스킬과 동일한 HTML `Audio` 경로로 `R1.mp3`를 재생합니다.
- **W 오디오 채널 전환**: W1·W2는 `W` 채널에서 단일 재생합니다. W2 시전 시 `W_CROSSFADE_MS=160` 동안 W1 볼륨을 줄이면서 W2 볼륨을 올린 뒤 W1을 정지합니다. W1·W2 루프와 W2 종료 페이드아웃은 적용하지 않습니다. 일반 과전하 진입·실험체 초기화로 W가 취소될 때 W 채널을 정지하며, R2 적중 과전하는 W 채널과 차징을 유지합니다.
- **R 오디오 채널 전환**: R1·R2는 `R` 채널에서 단일 재생합니다. R2 시전 시 `R_CROSSFADE_MS=160` 동안 재생 중인 R1 볼륨을 줄이면서 R2 볼륨을 올린 뒤 R1을 정지합니다. 실험체 초기화 시 R 채널을 정지합니다.
- **적중 사운드 분기**: W2의 3.25~3.75m `전하소산(경계)`=`W_타격.mp3`, 3.25m 미만 `전하소산(내부)`=`W_안.mp3`, E1·E2=`E_타격.mov`, R1·R2=`R_타격.mov`, 원거리 평타=`평타_원거리.mp3`, 원거리 Q=`Q_원거리_타격.mov`, 근거리 Q=`Q_타격.mov`를 재생합니다. 근거리 평타·D는 `타격.mov`를 유지하며 광역 다중 적중은 판정별 한 번 재생합니다.
- **S 키 = 이동 정지**: 진행 중이던 클릭 이동·추격 평타·버퍼된 이동 명령을 취소합니다. (돌진·R2 텔레포트 등 스킬 이동은 유지)
  - S는 정식 조작키 액션이라 **조작키 변경**에서 재지정할 수 있습니다.
- **스킬 사거리 표시 기본 꺼짐**: 첫 진입 시 "스킬 사거리 표시" 옵션이 OFF가 되어, 기본 조작이 즉시 시전(quickCast)입니다.
- **캐릭터 설정** 그룹 추가: 사이드 패널에 **공격 속도(회/초)** · **이동 속도(m/s)** 를 실시간 조절하는 입력란을 추가했습니다. (이동 속도 입력은 이쪽으로 이동)
- **이동 속도 표시**: 우측 상단에 현재 실효 이동 속도(m/s)가 표시됩니다.
- **A 활성 + 사거리 내부 좌클릭**: A로 평타 사거리 표시가 활성화된 상태에서 캐릭터의 현재 평타 사거리 원 내부를 좌클릭하면, 이동·추격 없이 사거리 안의 최근접 적에게 즉시 평타를 요청합니다. 사거리 원 바깥 좌클릭은 공격하지 않습니다.
- **스킬 후 자동 공격**: 설정을 켜면 스킬 시전·돌진·W 충전이 끝난 뒤 평타 사거리 안의 최근접 더미를 계속 자동 공격합니다. `basicAttack()`의 평타 쿨다운(`1 / 공격 속도`)을 그대로 적용하며, 이동·정지·수동 공격 명령 또는 대상의 사거리 이탈 시 종료됩니다. 사거리 복귀 후 자동으로 재개되지 않습니다.
- **전체화면 조작키 재지정**: 기본 F11은 Fullscreen API 전체화면 전환, 기본 Esc는 전체화면 종료입니다. 두 액션 모두 **조작키 변경** 모달에서 재지정할 수 있습니다. 키 입력 대기 중에는 Esc도 취소가 아닌 바인딩 키로 등록됩니다.
- **쿨타임 초기화 UI 정리**: 별도 "쿨타임 초기화" 버튼을 삭제했습니다. C 단축키로 토글하는 체크박스 표기를 "쿨타임 초기화"로 변경했습니다. 단축키 재지정은 조작키 변경에서 유지됩니다.
- **디버그(쿨타임 표시)**: F12 콘솔에서 `debug()`(토글) / `debug(true)` / `debug(false)` 로 켜고 끕니다. 켜면 우측 상단에 모든 스킬 쿨타임·재시전·전하/과전하 상태가 텍스트로 표시됩니다.
- **사이드 패널 정리**: 상단 "상태" 카드(상태/전하 슬롯/타이머)를 제거했습니다.

## 스킬 변경
- **W 풀차징 내부 판정**: 풀차징(`ratio >= 0.86`)이어도 기준 원 내부인 3.25m 미만은 `전하소산(내부)`입니다. 풀차징의 3.25~3.75m만 `전하소산(경계)`이며, 풀차징이 아니면 0~3.75m 전체를 `전하소산(내부)`로 판정합니다. 3.75m 초과는 빗맞음입니다.
- **원거리 Q 탄환 소모**: 전자포 사용 시 남은 탄환 수와 관계없이 과전하 탄환을 1개만 소모합니다.
- **W 음성 페어링**: W1 시전마다 `state.wVoiceIndex`에 `0..2` 중 하나를 저장하고 W1·W2에 같은 인덱스의 음성을 적용합니다. 수동 W2와 자동 방출 W2 모두 동일한 페어 음성을 재생하며, W2 방출·W 취소·실험체 초기화 시 인덱스를 해제합니다.
- **W1 최대 차징 유지시간 감소**: 완충 유지시간 `W_HOLD_FULL`을 2.5초에서 1.5초로 1초 단축했습니다. 기본 충전 0.8초를 포함한 자동 방출까지의 전체 창은 3.3초에서 2.3초로 감소합니다.
- **R 음성 분리·페어링**: R1 시전마다 `x=1,2,3` 중 하나를 랜덤 선택해 `src/voice/R/Aiden_skillR_x_01.wav`를 재생하고, 해당 R의 R2는 같은 `x`의 `Aiden_skillR_x-2_01.wav`를 재생합니다. 선택 인덱스는 `state.rVoiceIndex`에 저장하며 R2 사용·재시전 만료·초기화 시 해제합니다. R2 적중으로 과전하 음성이 발생해도 R2 처리 뒤 대사를 재생해 덮어쓰기를 방지합니다.
- **E 음성 분리**: E1은 `src/voice/E/Aiden_skillE1_01.wav`, E2는 대응 파일 `src/voice/E/Aiden_skillE1-2_01.wav`만 재생합니다. E1·E2가 서로의 파일을 랜덤 선택하지 않습니다.
- **과전하 패시브 음성**: 근접 폼에서 원거리 과전하 폼으로 실제 전환될 때만 `src/voice/passive`의 7개 음성 중 하나를 랜덤 요청합니다. 이미 다른 캐릭터 대사가 재생 중이면 패시브 대사 요청도 폐기되며, R2 재진입에는 요청하지 않습니다.
- **D(빗겨 흘리기)**: 적중해도 **전하(과전하 스택)를 쌓지 않습니다.**
- **D 대기·돌진 거리**: 시전 후 `CAST.D=0.75`초 동안 제자리에 고정되고, 대기 종료 후 조준 방향으로 정확히 3m를 돌진하며 경로상의 적에게 피해를 줍니다. 방어 연출 지속시간도 0.75초에 맞춥니다.
- **D 시전 중에는 R·E를 포함한 모든 스킬을 사용할 수 없습니다.** 또한 D 시전 중 **R2(낙뢰 이동)·E2(볼트 러시)는 선입력(버퍼)되지 않아** D가 끝난 뒤 의도치 않게 발동되지 않습니다.
- **F(전술 스킬)**: 사거리를 **3m**로 설정하고, 사거리 표시를 **원형**으로 변경했습니다.
- **F 좌표 순간이동**: 이동 보간(`dashTo`)을 제거하고 시전 완료 시 목적 좌표로 즉시 변경(`teleportTo`)합니다.
- **E1(백스텝)**: E 입력 순간의 마우스 월드 좌표를 `e1AimTarget`으로 고정합니다. `CAST.E1` 시전시간 종료 시점의 에이든 위치를 새 원점으로 삼아 `direction(state.aiden, e1AimTarget)`을 다시 계산하고, 해당 방향 0.5m 앞에서 탄환을 발사하며 반대 방향으로 백스텝합니다. 탄환은 `fireDir` 직선으로만 이동하고 명중 시 그 직선상의 지점에서 사라집니다. **명중 폭 0.6m**, 사용 후 **이동 속도 +20%** 버프(2초).
- **E 시전 사운드 분리**: 백스텝(E1) 시전 시작에는 `src/sound/E1.mov`, 볼트 러시(E2) 시전 시작에는 `src/sound/E2.mov`를 재생합니다. 두 사운드는 서로 교차 재생하지 않습니다.
- **E 적중 사운드**: 백스텝(E1) 또는 볼트 러시(E2)가 적에게 적중한 시점에 `src/sound/E_타격.mov`를 재생합니다. 빗맞은 E1에는 재생하지 않습니다.
- **근거리 Q / 원거리 Q 쿨타임 독립**: 두 Q는 완전히 별도 쿨다운(`Q_NORMAL`/`Q_OVER`)으로 돕니다. 각 Q는 **자기 쿨만 시작**하며 서로 연결되지 않습니다. 다만 **이미 돌고 있는 쿨은 폼이 바뀌어도 백그라운드로 계속 진행**됩니다 — 근거리 Q를 쓰고 과전하로 전환하면 근거리 Q 쿨이 내부에서 계속 돌고, 그 사이 원거리 Q는 자기 쿨이 0이면 바로 사용 가능합니다(반대도 동일). 평타 적중 시 Q 쿨 1.5초 감소는 현재 모드의 Q에만 적용됩니다.
- **E2(볼트 러시)**: **시전시간 없이 즉시** 발동(돌진)합니다.
- **E2 중 W2 허용**: W가 충전된 상태라면 볼트 러시(E2) 돌진 도중에도 W를 다시 눌러 전하 소산 2타를 즉시 방출할 수 있습니다.
- **R2(낙뢰 이동)**: **시전시간 없이 즉시** 발동하며 이동 보간 없이 낙뢰 좌표로 즉시 변경(`teleportTo`)됩니다.

## 콤보 (R 위빙)
- **R2 이전에 다른 스킬을 섞으면**, R 위치로 이동한 뒤 그 위치에서 섞은 스킬이 나갑니다.
  - **RQR**: R1 → Q → R2(이동) → 이동한 위치에서 Q 발동
  - **RER**: R1 → E → R2(이동) → 이동한 위치에서 E 발동
  - **RWR**: R1 → W → R2(이동) → 이동한 위치에서 W 발동
- **RER 불가 원인**: E1은 0.16초, R1은 0.34초입니다. R1 시전 중 E→R을 빠르게 입력하면 E1이 먼저 완료되고, R2는 R1 완료까지 입력 버퍼에 남았습니다. 따라서 E1이 순간이동 전 위치에서 발사된 뒤 R2가 실행됐습니다. 기존 방향 벡터 문제와 별개인 **캐스트 완료 순서** 문제였습니다.
- **RER 구현**: E 입력 시 **목표 월드 좌표**를 복사합니다. E1 완료 시 R2가 입력 버퍼에서 대기 중이면 `pendingE1Release`에 방출을 보류합니다. R1 완료 후 버퍼된 R2가 `teleportTo()`를 실행한 직후 `releaseE1()`을 호출하고, **순간이동한 에이든 좌표 → 저장된 목표 좌표**의 방향 벡터를 재계산합니다. R1 시전 중 입력한 빠른 R→E→R과 R2 준비 후 입력한 E→R을 모두 지원합니다.
- **RQR 유지**: Q는 시전 시작의 방향 벡터를 유지합니다. R2 이후 위치에서 같은 벡터로 발사되는 기존 RQR 조작을 변경하지 않습니다.
- **돌진 중 재시전 이동 차단**: `state.dash` 진행 중 E2(볼트 러시)·R2(낙뢰 이동)를 사용할 수 없습니다. E2 돌진 중 R2 입력도 무시하며, 돌진 종료 뒤 자동 발동하지 않도록 입력 버퍼에 저장하지 않습니다. 충전된 W2 방출 예외는 유지합니다.

## 버그 수정
- **W 차징 바 이동 정지 시 소멸**: 상태뿐 아니라 DOM도 `#wCastPanel`·`#wCastFill`·`#wCastName`으로 일반 캐스트 바와 분리했습니다. `renderWCast()`가 매 프레임 독립 렌더링하며 W1 시작부터 W2 방출까지 유지합니다. 이동 정지·S 입력·일반 캐스트 렌더링은 W 전용 DOM을 숨길 수 없습니다.
- **RW 시전 끊김/딜레이**: 긴 차단 시전(예: R1) 동안 입력 버퍼가 만료되어 다음 스킬(W 등)이 씹히던 문제를 수정 — 차단 시전 중 입력 버퍼를 유지해 시전이 끝나면 바로 이어집니다.
- **패시브(과전하) 종료 기믹**: 과전하는 **지속시간(7초) 종료** 또는 **탄환 소진** 시 끝나며, 종료 시 이동 속도 가속(+13%) 기믹이 적용됩니다.
- **전하 지속시간(5초)**: 전하는 획득 후 5초 동안 유지되며, **전하를 얻을 때마다 지속시간이 초기화**됩니다. 5초 동안 추가 획득이 없으면 전하가 소멸합니다. 전하가 가득 차 과전하가 되면 전하는 과전하(탄환)로 전환됩니다.
- **풀차지 시 백스텝 초기화**: 4스택에서 백스텝(E)으로 5스택을 채우면 백스텝 쿨이 초기화되고 볼트 러시(E2) 상태는 유지됩니다. **볼트 러시 사용 후에도 백스텝 쿨이 다시 걸리지 않아** 바로 백스텝(E1)을 한 번 더 쓸 수 있습니다.
- **WW 멈춤 딜레이**: W(전하 소산)는 이동하며 충전하는 스킬이라 **이동 정지/이동 명령 취소 대상에서 제외**했습니다. 이동 중 W를 눌러도 이동이 끊기지 않습니다.
- **일반 과전하 진입 시 W 쿨 초기화 버그**: 거리 조건·백스텝 등 일반 과전하 진입으로 충전 중 W가 취소될 때 쿨다운이 적용되지 않아 W가 무료로 재사용되던 문제를 수정. R2 적중 과전하는 W를 취소하지 않습니다.
- **R2 적중 시 W 차징 소멸 수정**: 원인은 R2 적중이 `enterOvercharge(..., true)`를 호출하고, 과전하 진입 공통 로직이 `wStart`·`recast.W`·`wChargeHudActive`·W 음성 인덱스를 초기화한 것입니다. `enterOvercharge`에 `preserveW` 옵션을 추가하고 R2 적중 경로에서만 활성화해 차징 진행도·차징 바·W1 사운드·W 음성 페어를 유지합니다.
- **W 사거리 표시**: 이동 중이 아니어도, W 시전(충전) 시작 시점부터 W 범위가 보이도록 했습니다.
- **Q 쿨 오적용 수정**: 시전 중에 모드가 바뀌면(원거리 Q가 마지막 탄환 소진으로 과전하 종료, 또는 근거리 Q가 풀차지를 채워 과전하 진입) 쿨이 **반대 폼의 Q에 잘못 걸리던** 문제를 수정. Q 폼(근/원)과 그 쿨 적용 대상을 **시전 시작 시점에 고정**해, 시전 중 모드 변화와 무관하게 항상 올바른 폼(`Q_NORMAL`/`Q_OVER`)에 쿨이 적용됩니다.
- **과전하 중 R2 보상 미적용 수정**: 과전하 상태에서 탄환을 일부 쓴 뒤 R2(낙뢰 이동)를 적중시켜도 **탄환 충전·E(백스텝) 초기화가 안 되던** 버그.
  - 원인: R2 적중 보상이 `state.charge = MAX` + `resetECooldownOnFullCharge(before)`였는데, 과전하 중에는 `state.charge`가 이미 MAX(5)라 ① 탄환(`state.bullets`)은 안 채워지고 ② `before < MAX_CHARGE` 조건이 거짓이 되어 E 초기화가 안 됐다.
  - 수정: **R2를 시전해 낙뢰가 적에게 맞으면 과전하 즉시 (재)돌입**. `castSecondLightning()`이 적중 여부를 반환하고, 수동 R2 시전이 적중하면 `state.charge = MAX` 후 `enterOvercharge("낙뢰 과전하", true)` 호출. `enterOvercharge`에 `force` 인자를 추가해 이미 과전하여도 재돌입(탄환·지속시간 재충전)하고, **과전하 진입의 부수효과로 백스텝(E)을 초기화**(`enterOvercharge` 안에서 `resetBackstep()`).

- **더미 히트박스**: 더미는 히트 판정에서 점으로 처리됐는데, **더미 시각 크기의 약 0.9배 히트박스 반경(≈24px)** 을 부여해 스킬이 더미 중심을 정확히 안 맞아도 몸에 닿으면 적중하도록 했습니다.

## 참고
- 공격 속도/이동 속도는 "캐릭터 설정"에서 실시간으로 바꿀 수 있습니다(공격 속도는 근접·원거리 공통 회/초).
- 좌상단이 아닌 **우상단**에 현재 이동 속도(m/s)가 표시됩니다.
